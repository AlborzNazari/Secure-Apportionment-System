# Contributing Guidelines

Thank you for considering contributing to the Secure Parliamentary Seat Apportionment System! We welcome contributions from the community to help improve this project.

## How to Contribute

### Reporting Bugs
- Use the [Issues](https://github.com/AlborzNazari/Secure-Apportionment-System/issues) tab to report bugs.
- Check if the issue already exists before creating a new one.
- Provide a clear title, description, steps to reproduce, expected vs. actual behavior, and screenshots if applicable.

### Suggesting Features
- Use the Issues tab with a "feature request" label.
- Explain the problem you're trying to solve and why the feature would be useful.

### Submitting Pull Requests (PRs)
1. Fork the repository and create a new branch for your changes: `git checkout -b feature/your-feature-name`.
2. Make your changes and commit them with clear messages.
3. Ensure your code follows the project's style (PEP 8 for Python).
4. Add or update tests if applicable (run `pytest` to verify).
5. Push to your fork and submit a PR to the `main` branch.
6. Describe your changes in the PR description, reference any related issues.

### Code Style
- Follow PEP 8 guidelines.
- Use meaningful variable/function names.
- Add docstrings for functions and modules.
- Keep code modular and readable.

### Testing
- All changes should pass existing tests.
- Add new tests for new features or bug fixes.

### Documentation
- Update README.md or docs/ files if your changes affect usage.
- For major changes, add to CHANGELOG.md (if it exists).
